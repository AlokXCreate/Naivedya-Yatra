NaivedyaYatra: Technical Project Report

An AI-Powered Cultural Preservation Platform for Traditional Indian Culinary Heritage

Author: Alok Kadam (ALoKX)  |  Date: June 15, 2026  |  Status: Production Ready

1. Executive Summary

NaivedyaYatra is an immersive, client-side Single-Page Web Application (SPA) designed to archive, scale, and filter 52 authentic regional Indian recipes, emphasizing sacred temple offerings (Prasad/Bhog), festival diets, and Ayurvedic nutritional parameters.

Built using a bespoke modern-traditional Indian design system, the application includes a dark-mode responsive UI themed with rich spiritual maroons, sandalwood paper canvases, and glowing marigold highlights. It incorporates browser-level capabilities to create a grease-free, kitchen-safe cooking experience:

Hands-Free Gesture Interface

Utilizes webcam frame-differencing pixel algorithms to navigate cooking instructions with a simple wave, keeping phone screens clean.

Multilingual Vocal Assistant

Continuous-loop speech recognition supporting 8 major Indian languages for navigations and step-by-step vocal feedback.

Non-Linear Portion Scaler

Applies food-science batch curves to throttle trace leaveners, intense spices, and water evaporation ratios up to 50 servings.

Offline Architecture First

The platform runs completely client-side. Local data persistency is managed using IndexedDB, meaning users can view, filter, bookmark, and cook recipes even in low-reception basements or remote kitchens.

2. Project Context & Motivation

Indian culinary history is intimately tied to regional geography, temples, seasonal agriculture, and scriptural guidelines. Many traditional dishes, especially Maha Prasad (temple offerings) and strictly coded dietary meals (Sattvic, Jain, Vrat/fasting diets), are at risk of vanishing as younger generations adopt standardized fast foods.

Additionally, modern cooking websites present serious usability issues in actual kitchen environments:

  • Physical Contamination: Wet, flour-dusted, or oil-slicked fingers smear mobile touchscreens, risking damage to expensive devices.
  • Inaccurate scaling: Standard apps double all ingredients linearly when scaling from 2 to 20 portions, resulting in over-salted, heavily spiced, or liquid-logged failures.
  • Scriptural Ignorance: General recipes do not automatically account for root-vegetable exclusions, grain replacements during sacred fasts, or mineral-salt conversions.

NaivedyaYatra targets these exact pain points by integrating advanced browser APIs, non-linear mathematical models, and dietary rule engines to build a secure, cultural, and smart digital kitchen assistant.

A library of 52 verified traditional recipes is categorized into regional tracks: Sacred Bhog & Prasad, Western Trails, South Indian Agraharams, Northern Kadhais, and Eastern Sweets & Rituals. Instant filter chips sort recipes by region, cooking method (Clay Pot, griddle, slow cook, griddle), or occasion (Vrat, Fasting, Prasad, Festivals). A phonetic search engine matches common variations, so typing "batata" or "aloo" yields correct potato-based items.

This mode triggers an enlarged parchment overlay optimized for visibility from distance. It houses the hands-free webcam wave sensors, continuous multi-lingual voice recognition (Hindi, Marathi, English, etc.), and the Tibetan Singing Bowl timer alarm synthesized using raw oscillator nodes on the Web Audio API (emitting frequencies at 220Hz, 440Hz, 560Hz, 680Hz, and 880Hz decaying exponentially over 5 seconds).

4. System Architecture & Data Flows

NaivedyaYatra is built purely with web-native technologies, avoiding bulky server runtimes to ensure rapid loading speeds and full static hosting eligibility.

System Architecture Block Diagram

Figure 1: High-level topology diagram of NaivedyaYatra showing client views, SPA router, storage layer, and media APIs. Click to enlarge.

A. Client-Side Hash Router

The application implements a custom routing system in src/router/router.js. Hash URL changes (e.g., #/recipes or #/guru) are caught by window listeners, clearing parent document grids and painting the respective page view dynamically. This prevents state loss in the ongoing voice recognition session or chatbot cache.

B. Client-Side IndexedDB Storage Topology

To operate reliably offline, the database layer (src/db/db.js) sets up a local IndexedDB object store structure (RasoiParamparaDB - Version 3):

  • recipes (KeyPath: id): Holds the 52 seeded recipes with metadata, ingredient lists, and local photo asset links.
  • users (KeyPath: email): Holds credentials, names, and customized dietary parameters.
  • user_recipes (KeyPath: id): Manages bookmarks, custom lists (e.g., "Monday Vrat Meals"), and personal notes.
  • reviews (KeyPath: id): Holds community ratings, text reviews, and user-uploaded base64 recipe photos.

C. Media APIs Integration

The system connects to three separate browser APIs:

  • Web Speech API (SpeechRecognition): Runs continuously. When voice recognition closes due to browser timeouts, a restart hook re-initializes it. Custom string match vectors capture navigational phrases in 8 regional languages.
  • SpeechSynthesis (TTS): Voices active recipe instructions aloud. Voices are programmatically filtered to Indian locales (e.g., hi-IN, en-IN) for natural phonetics.
  • Web Audio API: Triggers when step timers reach zero. Rather than loading an external MP3 file, it dynamically creates oscillator nodes to produce a bell chime with rich harmonics.

5. Mathematics & Algorithms

A primary differentiator of NaivedyaYatra is its departure from linear scaling in favor of culinary mathematics, along with its pixel-level motion vector algorithm.

A. Non-Linear Serving Multiplier Math

Scaling a recipe linearly (e.g., multiplying salt and cardamom by 10x for 10x servings) ruins taste. Evaporation rates, chemical reactions, and surface area ratios dictate sub-linear adjustments. The application defines distinct exponents based on ingredient category:

1. Trace Leaveners & Intense Spices (Salt, Camphor, Baking Soda)

Scaled using an exponent of \(0.55\) to prevent chemical toxicity and over-salting:

$$\text{Multiplier}_{\text{trace}} = (\text{Base Multiplier})^{0.55}$$

Example: For \(50\text{ servings}\), the multiplier is \(50^{0.55} \approx 8.6\text{x}\) (not \(50\text{x}\)).

2. Aromatic Ground Spices & Herbs (Cardamom, Cumin, Ginger, Pepper)

Scaled using an exponent of \(0.72\) to balance volatile oils:

$$\text{Multiplier}_{\text{spices}} = (\text{Base Multiplier})^{0.72}$$

Example: For \(50\text{ servings}\), spices scale by \(50^{0.72} \approx 16.7\text{x}\) (not \(50\text{x}\)).

3. Evaporating Liquids (Water, Broth, Coconut Milk)

Scaled using an exponent of \(0.85\) as larger vessels exhibit lower relative evaporation ratios:

$$\text{Multiplier}_{\text{liquids}} = (\text{Base Multiplier})^{0.85}$$

Example: For \(50\text{ servings}\), liquids scale by \(50^{0.85} \approx 27.8\text{x}\).

4. Bulk Caloric Bases (Rice, Flour, Lentils, Sugar, Ghee)

Scaled linearly (\(1.0\)) to preserve structural yields and calorie counts:

$$\text{Multiplier}_{\text{base}} = \text{Base Multiplier}$$

Example: For \(50\text{ servings}\), flour scales by exactly \(50.0\text{x}\).

B. Webcam Motion Detection Algorithm

To avoid overhead, the motion tracker uses basic canvas pixel diffs instead of heavy machine learning libraries (like TensorFlow):

  1. Every frame, the webcam feed is downsampled to a hidden \(40 \times 30\) canvas.
  2. For each cell \((x,y)\), red, green, and blue values are converted to luminance \(Y\):
    $$Y(x, y) = 0.299 \cdot R(x, y) + 0.587 \cdot G(x, y) + 0.114 \cdot B(x, y)$$
  3. We compute the absolute frame-to-frame difference for each cell:
    $$\Delta Y(x, y) = |Y_{\text{current}}(x, y) - Y_{\text{previous}}(x, y)|$$
  4. If \(\Delta Y(x, y)\) exceeds \(0.10\) (10% shift), the cell is flagged as active. If a cohesive cluster of active cells shifts horizontally from left to right across sequential frames, a hand-wave is registered, advancing the page instructions. A 2.0-second cooldown is enforced to prevent accidental duplicate triggers.

7. Installation & Setup Guide

Follow these steps to configure and run the NaivedyaYatra codebase locally on your machine:

1. Clone the Repository

git clone https://github.com/AlokXCreate/naivedya-yatra.git
cd naivedya-yatra

2. Install Development Dependencies

Ensure Node.js is installed on your machine, then run:

npm install

3. Start the Local Server

npm run dev

The application will start, usually serving on http://localhost:5173/. Open this URL in a Google Chrome or Microsoft Edge browser for full webcam and Speech API compatibility.

4. Build for Production

To compile optimized assets for web deployment (e.g. Netlify, Vercel):

npm run build

The compiled, production-ready static assets will be output to the dist/ folder.
